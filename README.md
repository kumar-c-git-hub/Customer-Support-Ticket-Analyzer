📊 Customer Support Ticket Analysis
A self driven Python project analyzing real world style customer support tickets to turn raw ticket data into actionable insights for operations and customer experience.

🎯 Objective
•	Explore how support tickets reflect customer pain points and expectations.
•	Identify patterns in priority, response behavior, and recurring themes (e.g., “slow response”, “delay”).
•	Provide a simple but repeatable pipeline for text based support ticket analysis.

🧩 What I did
•	Performed end to end analysis of 12 customer support tickets using Python.
•	Built a streamlined pipeline for:
•	Data cleaning and preprocessing
•	Ticket categorization (priority, status, channel)
•	Text based theme and sentiment pattern extraction
•	Summarized key metrics and patterns using descriptive statistics and text summaries (no advanced visualizations).

🔍 Key Insights
•	High priority tickets form a significant share of feedback, indicating urgent issues that need attention.
•	Keywords like “slow response” and “delay” appear frequently, pointing to time to fix gaps.
•	Positive experiences are present but often generic, whereas negative feedback tends to be more specific.
•	A small subset of tickets drives most of the pain points, suggesting targeted follow up can have a big impact.

⚙️ Tech Stack
•	Python for analysis and automation
•	Pandas for data cleaning and manipulation
•	Basic text analysis and statistical techniques (token counting, keyword extraction, simple summaries)

📌 Conclusion
This project shows how even a small set of customer support tickets can reveal meaningful patterns when analyzed systematically. By combining data cleaning, text based theme extraction, and descriptive statistics, I was able to highlight urgent issues, response time gaps, and areas where customer experience can be improved. The same pipeline can be scaled to larger datasets and extended with dashboards or visualizations in the future.
